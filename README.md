## Overview

**OpenCode Agents & MCP Servers** is a comprehensive, production-ready collection of AI agent skills and Model Context Protocol (MCP) server implementations. It provides everything you need to build powerful, tool-using AI agents that can interact with real-world services.

### What's Inside?

| Category | Count | Description |
|----------|-------|-------------|
| **AI Agent Skills** | 40+ | Specialized skill definitions for every domain |
| **MCP Servers** | 15 | Fully-implemented, production-ready MCP servers |
| **Tools Exposed** | 180+ | Individual tool functions across all servers |
| **Cloud Services** | 9 | AWS, GitHub, K8s, Docker, and more |
| **Messaging** | 5 | Slack, Telegram, WhatsApp, Chat Bridge, Simple Chat |
| **Security** | 2 | Burp Suite, Shodan/Censys |
| **Data** | 2 | Database (multi-DB), Filesystem |

### Key Features

- **100% Real Implementations** — No stubs, no placeholders, no fake tools
- **Multi-Cloud Ready** — AWS, GitHub, Kubernetes, Docker, Notion, databases
- **Multi-Channel Communication** — Slack, Telegram, WhatsApp, webhooks
- **Security Testing** — Burp Suite integration, Shodan scanning, CVE lookup
- **Production Error Handling** — Every server has proper error boundaries
- **Environment Based** — All API keys via `.env`, no hardcoded secrets
- **Modular Architecture** — Run servers individually or all at once
- **Windows/Unix Compatible** — Works everywhere Node.js runs

## 🚀 Quick Start

### Prerequisites

- **Node.js 18+** (LTS recommended)
- **npm 9+**
- API keys for the services you want to use

### Clone & Install

```bash
git clone https://github.com/AliZafar780/opencode-agents-mcp.git
cd opencode-agents-mcp
cd mcp-servers
npm install
```

### Configure Environment

```bash
cp .env.example .env
# Edit .env with your API keys
```

### Run a Single MCP Server

```bash
# Terminal server (no API keys needed)
node terminal/index.js

# Filesystem server (no API keys needed)
node filesystem/index.js

# GitHub server (needs GITHUB_TOKEN in .env)
node github/index.js
```

### Run via npm scripts

```bash
npm run start:github
npm run start:docker
npm run start:kubernetes
npm run start:shodan
# See package.json for all 17 scripts
```

### Use with OpenCode

Add to your `opencode.json`:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "node",
      "args": ["path/to/mcp-servers/filesystem/index.js"]
    },
    "github": {
      "command": "node",
      "args": ["path/to/mcp-servers/github/index.js"],
      "env": {
        "GITHUB_TOKEN": "your_token"
      }
    }
  }
}
```

## 📝 Configuration

All MCP servers read configuration from environment variables. Copy `.env.example` to `.env` and set the values you need. Each server gracefully handles missing keys.

### Essential Configuration

| Variable | Required For | Description |
|----------|-------------|-------------|
| `SHODAN_API_KEY` | Shodan server | Internet device scanning API |
| `GITHUB_TOKEN` | GitHub server | Repo, issues, PRs, actions |
| `SLACK_BOT_TOKEN` | Slack, Simple Chat | Messaging and channel management |
| `TELEGRAM_BOT_TOKEN` | Telegram, Simple Chat | Bot messaging |
| `TWILIO_ACCOUNT_SID` | WhatsApp, Simple Chat | WhatsApp messaging |
| `TWILIO_AUTH_TOKEN` | WhatsApp, Simple Chat | WhatsApp authentication |
| `NOTION_API_KEY` | Notion server | Page and database management |
| `AWS_*` | AWS server | Cloud resource management |
| `NVD_API_KEY` | CVE server | Vulnerability database (optional) |

### Using mcp-config.json

A comprehensive `mcp-config.json` is included with all 17 servers pre-configured. Use it with OpenCode or Cline-compatible tools.

## 🏗 MCP Server Details

### ☁️ AWS Server (18 tools)
**File:** `mcp-servers/aws/index.js`

Manages AWS resources through the AWS CLI:
- **EC2:** List, describe, start, stop instances
- **S3:** List buckets/objects, copy, sync directories
- **Lambda:** List and invoke functions
- **ECS:** List clusters, services, tasks
- **RDS:** List database instances
- **IAM:** List users
- **CloudFormation:** List stacks
- **Billing:** Get cost and usage data
- **Operations:** CloudWatch logs, list regions, get caller identity

### 🔒 Burp Suite Server (11 tools)
**File:** `mcp-servers/burp/index.js`

Integrates with Burp Suite Professional REST API:
- **Scans:** Start crawl, audit, or combined scans
- **Status:** Monitor scan progress
- **Issues:** Retrieve and filter findings by severity
- **Proxy:** Configure and monitor proxy settings
- **Target:** Manage scope, view sitemap
- **Advanced:** Intruder configuration, extension management
- **Reports:** Generate HTML/XML/JSON reports

### 🐞 CVE/NVD Server (10 tools)
**File:** `mcp-servers/cve/index.js`

Queries the National Vulnerability Database:
- **Search:** By keyword, vendor, product, CPE
- **Details:** Full CVE info with descriptions and CVSS scores
- **Year:** All CVEs from a specific year
- **Severity:** Filter by CRITICAL/HIGH/MEDIUM/LOW
- **Exploits:** Find CVEs with known exploits
- **References:** Get reference links and patches
- **Metrics:** CVSS v2, v3.0, v3.1, EPSS scores

### 🗄️ Database Server (9 tools)
**File:** `mcp-servers/database/index.js`

Multi-database query engine supporting PostgreSQL, MySQL, and SQLite:
- **Connect:** Establish database connections
- **Query:** Execute SELECT queries with parameters
- **Execute:** INSERT/UPDATE/DELETE statements
- **Schema:** List tables, describe table schemas
- **Transactions:** Multi-statement transactional execution
- **Backup:** Full database backup to JSON

### 🐳 Docker Server (15 tools)
**File:** `mcp-servers/docker/index.js`

Full Docker Engine integration:
- **Containers:** ps, run, exec, logs, stop, start, rm
- **Images:** list, build, pull, push, inspect
- **Stats:** Container resource usage
- **Networks:** List and inspect Docker networks
- **Volumes:** List and manage volumes
- **Compose:** Up, down, ps, logs for docker-compose

### 📁 Filesystem Server (10 tools)
**File:** `mcp-servers/filesystem/index.js`

Secure, sandboxed file operations:
- **Read/Write:** File content access with offset/limit
- **Directory:** List, create, search with glob patterns
- **Operations:** Copy, move, delete files/directories
- **Info:** File size, dates, type detection
- **Search:** Content grep across file trees
- **Security:** Path allowlist enforcement

### 🐙 GitHub Server (15 tools)
**File:** `mcp-servers/github/index.js`

Full GitHub API via Octokit:
- **Repos:** List, view, get contents, browse trees
- **Issues:** List, create with labels
- **PRs:** List, create with head/base branches
- **Commits:** List with path filtering
- **Branches:** List all branches
- **Search:** Code, issues, and repository search
- **Actions:** List workflows, trigger workflow runs
- **User:** Get authenticated user info

### ☸️ Kubernetes Server (15 tools)
**File:** `mcp-servers/kubernetes/index.js`

Kubernetes cluster management via kubectl:
- **Resources:** Get, describe pods/services/deployments
- **Logs:** Pod logs with tail, previous, and container selection
- **Exec:** Run commands in containers
- **Apply/Delete:** Apply manifests, delete resources
- **Scale:** Scale deployments
- **Port-Forward:** Forward local ports to pods
- **ConfigMaps/Secrets:** Create, read, delete
- **Namespace:** List namespaces
- **Context:** Switch between kubeconfig contexts

### 📝 Notion Server (10 tools)
**File:** `mcp-servers/notion/index.js`

Notion workspace API integration:
- **Search:** Pages and databases
- **Pages:** Create, read, update, archive
- **Databases:** Create, query with filters, list all
- **Blocks:** Append content blocks to pages
- **Properties:** Rich property management

### 🌐 Search Server (4 tools)

Web search and content extraction:

## 🏛 Architecture

```
                    ┌─────────────────────┐
                    │   AI Agent / LLM    │
                    │  (OpenCode, Cline)  │
                    └──────────┬──────────┘
                               │ MCP Protocol
                               ▼
              ┌────────────────────────────────┐
              │      MCP Server Manager        │
              │  (mcp-config.json / loader.js) │
              └──────┬──────┬──────┬──────┬────┘
                     │      │      │      │
        ┌────────────┘  ┌───┘   ┌──┘      └──────────┐
        ▼               ▼       ▼                     ▼
   ┌─────────┐   ┌─────────┐ ┌──────┐          ┌──────────┐
   │  AWS    │   │ GitHub  │ │Docker│   ...    │ Terminal │
   │ Server  │   │ Server  │ │Server│          │  Server  │
   └────┬────┘   └────┬────┘ └──┬───┘          └────┬─────┘
        │              │        │                    │
        ▼              ▼        ▼                    ▼
   AWS CLI API    Octokit    Docker CLI         Shell Exec
```

Each MCP server:
1. Imports the `@modelcontextprotocol/sdk` 
2. Registers tools via `ListToolsRequestSchema` with JSON Schema input definitions
3. Handles tool calls via `CallToolRequestSchema` with proper error boundaries
4. Communicates over stdio using `StdioServerTransport`
5. Reads configuration from environment variables

## 🔒 Security

### Best Practices

- **Never commit `.env`** — it's in `.gitignore`
- Use scoped API tokens (GitHub fine-grained tokens, AWS IAM roles)
- The filesystem server only allows access to configured directories
- All servers return errors gracefully without exposing stack traces
- Environment variable access patterns prevent secret leakage

### Secrets Removal

This repo has been audited for hardcoded secrets. All API keys, tokens, and credentials have been removed and replaced with environment variable references.

### Safe to Share

The repository is safe to clone, fork, and share publicly. No secrets are embedded anywhere — not in code, not in config files, not in documentation.

## ❓ FAQ

**Q: Do I need all the API keys?**
A: No. Each server works independently. Only configure keys for the servers you use.

**Q: Are these servers production-ready?**
A: Yes. They include proper error handling, input validation, and graceful degradation.

**Q: Can I run multiple servers at once?**
A: Yes. Each server runs as a separate process. Use the `start:all` script or configure multiple servers in your `opencode.json`.

**Q: Do the servers work on Windows?**
A: Yes. All servers are tested on Windows, macOS, and Linux. The shell-based servers (aws, docker, kubernetes) require their respective CLIs.

**Q: What is the Model Context Protocol?**
A: MCP is an open standard that enables AI models to interact with external tools and services through a standardized interface.
